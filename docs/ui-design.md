# UI设计方案

## 一、设计理念

### 1.1 设计原则

**大屏优先**
- 主要针对平板设备（7-10寸）
- 支持横屏和竖屏
- 字体和按钮尺寸优化

**简洁易用**
- 一级功能突出展示
- 操作步骤最少化
- 信息层次清晰

**视觉友好**
- 温馨的配色方案
- 适当的留白
- 柔和的圆角和阴影

**家庭氛围**
- 使用可爱的图标
- 温暖的色彩
- 鼓励性的文案

## 二、视觉设计

### 2.1 配色方案

**主色调**
```
主色（Primary）: #6366F1 (柔和的蓝紫色)
├─ 用于：主要按钮、链接、强调元素
└─ 变体：
   ├─ Light: #818CF8
   ├─ Darker: #4F46E5
   └─ Soft: #E0E7FF (背景)

辅助色（Secondary）: #10B981 (清新的绿色）
├─ 用于：成功状态、完成标识
└─ 变体：
   ├─ Light: #34D399
   └─ Soft: #D1FAE5

警告色（Warning）: #F59E0B (温暖的橙色)
├─ 用于：警告提示
└─ 变体：
   ├─ Light: #FBBF24
   └─ Soft: #FEF3C7

错误色（Error）: #EF4444 (柔和的红色)
├─ 用于：错误、危险操作
└─ 变体：
   ├─ Light: #F87171
   └─ Soft: #FEE2E2
```

**中性色**
```
文本色：
├─ 主文本: #1F2937 (深灰)
├─ 次文本: #6B7280 (中灰)
├─ 辅助文本: #9CA3AF (浅灰)
└─ 禁用文本: #D1D5DB (禁用灰)

背景色：
├─ 主背景: #FFFFFF (纯白)
├─ 次背景: #F9FAFB (浅灰)
├─ 卡片背景: #FFFFFF (纯白 + 阴影)
└─ 输入框背景: #F3F4F6 (浅灰)

边框色：
├─ 主边框: #E5E7EB (浅灰)
├─ 分割线: #E5E7EB

```

**疲劳度颜色编码**
```
0.0-0.3: 轻松 😊
└─ 颜色: #10B981 (绿色)

0.3-0.5: 适中 😐
└─ 颜色: #F59E0B (橙色)

└─ 颜色: #F97316 (深橙)

0.8+: 过度疲劳 😫
└─ 颜色: #EF4444 (红色)
```

### 2.2 字体系统

**字体族**
```css

  -apple-system,
  BlinkMacSystemFont,
  "Segoe UI",
  "PingFang SC",
  "Hiragino Sans GB",
  "Microsoft YaHei",

```

**字号规范**
```

```

```
Medium:    500 (强调)
Semibold:  600 (小标题)
```

```
lg:  24px  (大间距)
xl:  32px  (区块间距)

```

**卡片内边距**
```

```

**圆角**
```

```


## 六、交互设计

### 6.1 任务操作流程

**完成任务**
```
1. 点击任务卡片右侧的 "✓" 按钮
2. 按钮变为加载状态 (旋转动画)
3. 任务卡片淡出，半透明显示
4. 系统重新计算疲劳度
5. 更新统计数据
6. 显示成功提示 "任务已完成！"
```

**拖拽分配任务**
```
1. 长按任务卡片 (300ms)
2. 卡片浮起，阴影加深
3. 拖动到父母头像/卡片上
4. 目标高亮显示 (边框 + 背景)
5. 松开，完成分配
6. 卡片更新显示新的分配人
7. 显示提示 "任务已分配给妈妈"
```

**快速添加任务**
```
1. 点击 "➕ 快速添加任务"
2. 弹出简洁表单:
   ┌─────────────────────────────┐
   │  添加任务                  │
   │  ────────────────────────   │
   │  名称: [____________]      │
   │  分类: [选择 ▼]           │
   │  时段: [选择 ▼]           │
   │  分配: [选择 ▼]           │
   │                            │
   │        [取消] [确认添加]   │
   └─────────────────────────────┘
3. 填写必要信息
4. 点击确认，关闭弹窗
5. 任务出现在今日任务列表
```

### 6.2 一键重新分配

**触发方式**
- 点击首页的 "一键重新分配" 按钮
- 点击任务页面的 "重新分配" 按钮

**交互流程**
```
1. 点击按钮
2. 显示确认对话框:
   ┌─────────────────────────────────┐
   │  重新分配任务                    │
   │  ────────────────────────────   │
   │  系统将根据当前负载重新分配     │
   │  所有未完成的任务。             │
   │                                 │
   │  原因: [可选填]                 │
   │                                 │
   │  预计影响:                       │
   │  • 将调整 7 个未完成任务        │
   │  • 妈妈疲劳度: 0.35 → 0.55      │
   │  • 爸爸疲劳度: 0.25 → 0.45      │
   │                                 │
   │         [取消] [确认重新分配]   │
   └─────────────────────────────────┘
3. 点击确认
4. 显示加载动画
5. 任务卡片依次更新分配人
6. 统计数据更新
7. 显示完成提示 "已重新分配7个任务"
```

### 6.3 手势操作

**滑动切换日期**
```
左滑: 下一页 (明天)
```

## 七、动画和过渡

### 7.1 页面切换

**淡入淡出**
```css
.fade-enter-active,
.fade-leave-active {
  transition: opacity 0.3s ease;
}

.fade-enter-from,
.fade-leave-to {
  opacity: 0;
}
```

**滑动切换**
```css
.slide-enter-active,
.slide-leave-active {
  transition: all 0.3s ease;
}

.slide-enter-from {
  transform: translateX(100%);
}

.slide-leave-to {
  transform: translateX(-100%);
}
```

### 7.2 任务状态变化

**完成任务**
```
1. 卡片背景变绿 (0.2s)
2. 卡片淡出 (0.3s)
3. 列表重新排列 (0.3s)
```

**加载状态**
```
1. 按钮显示旋转图标
2. 列表项显示骨架屏
3. 内容淡入显示
```

### 7.3 微交互

**按钮点击**
```
Hover: 轻微放大 (scale 1.05)
Active: 按下效果 (scale 0.95)
Focus: 主色边框 + 阴影
```

**卡片悬停**
```
Hover: 阴影加深 + 轻微上浮 (translateY -2px)
过渡: 0.2s ease
```

**拖拽反馈**
```
1. 拖动开始: 卡片浮起 (scale 1.05)
2. 拖动中: 跟随手指/鼠标
3. 放置目标: 高亮显示
4. 放置成功: 绿色闪烁
5. 放置失败: 红色闪烁 + 回弹动画
```
