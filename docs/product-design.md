# 产品设计方案

## 一、产品定位

### 1.1 产品愿景
> 让育儿不再焦虑，让家庭协作更高效，让每个成员都能得到充分休息。

### 1.2 目标用户
- **主要用户**：有0-3岁宝宝的夫妻家庭
- **使用场景**：家庭共享设备（平板/大屏）
- **核心痛点**：
  - 育儿任务繁杂，容易遗漏
  - 夫妻分工不均，一方过度劳累
  - 缺乏科学育儿知识指导
  - 睡眠不足，影响身心健康

### 1.3 产品价值观
- **减负优先**：自动化处理，减少手动操作
- **科学育儿**：基于权威知识库，而非道听途说
- **灵活调整**：适应家庭实际情况，不强制执行
- **家庭协作**：促进夫妻分工，共同承担责任

### 2.2 功能模块详解

#### 模块1：宝宝档案管理

**功能描述**
- 创建和编辑宝宝基本信息
- 记录生长数据（体重、身高、头围）
- 设置重要提醒（体检、疫苗等）

**核心字段**
```
- 基本信息：姓名、生日、性别、照片
- 生长数据：日期、体重、身高、头围、备注
- 特殊情况：过敏史、疾病史、特殊需求
```

**用户故事**
```
作为新手父母
我想记录宝宝的生长数据
以便追踪发育情况和咨询医生

作为父母
我想设置重要日期提醒
以便提前准备体检和疫苗接种
```

**功能描述**
- 添加父母及其他照护者信息
- 设置每个人的时间约束

**核心字段**
```
- 基本信息：姓名、关系、照片
```

```
- 1月：乙肝疫苗2、卡介苗

- 3月：百白破1、脊灰疫苗2、肺炎13价2
- 4月：百白破2、脊灰疫苗3、肺炎13价3
- 5月：百白破3
- 6月：乙肝疫苗3、流脑A群1
- 8月：麻腮风1、乙脑减毒活疫苗1
- 9月：流脑A群2
- 12月：肺炎13价4、水痘疫苗
- 18月：百白破4、麻腮风2、甲肝减毒活疫苗1
- 2岁：乙脑灭活疫苗2
- 3岁：流脑A+C群1
- 4岁：脊灰疫苗4、水痘疫苗2
```

##### 3.2 辅食添加指南
```
基于：中国营养学会《7-24月龄婴幼儿喂养指南》

月龄 → 辅食清单
- 6个月：强化铁米粉、蔬菜泥（菠菜、南瓜）、水果泥（苹果、香蕉）
- 7个月：肉泥（鸡肉、猪肉）、蛋黄、烂面条、粥
- 8个月：手指食物、肉末、碎菜、蛋黄饼
- 9个月：软饭、小块水果、自制饼干
- 10-12个月：家庭饭菜（少盐少糖）、全蛋、酸奶
- 12月+：过渡到家庭饮食，注意营养均衡
```

##### 3.3 早教游戏推荐
```
基于：《美国儿科学会》发育里程碑

月龄 → 早教活动

  - 俯卧抬头练习
  - 追视训练（黑白卡、彩色玩具）

- 6-9个月：
  - 爬行练习
  - 拇食指捏取
  - 积木游戏
  - 认知卡片
  - 听指令游戏

- 9-12个月：
  - 站立和扶走
  - 容器游戏（倒出/放入）
  - 简单拼图
  - 模仿游戏
  - 语言启蒙（指认、命名）
```

#### 模块4：智能任务调度

**功能描述**
- 根据月龄自动生成每日任务清单
- 结合时间线智能分配任务
- 支持手动调整和重新分配

**任务分类**

##### 4.1 固定任务
```
特点：时间固定，不可调整
- 喂奶/吃饭（按喂养频率）
- 换尿布（按需）
- 睡觉（白天小睡、夜间长睡）
```

##### 4.2 灵活任务
```
特点：可调整时段，但需完成
- 辅食制作
- 早教游戏
- 户外活动
- 清洁消毒（奶瓶、玩具）
```

##### 4.3 一次性任务
```
特点：特定日期，完成后删除
- 疫苗接种
- 体检
- 发育评估
- 补充维生素D等
```

**时间线设计**

```
一天6个时间段：
┌─────────────┬──────────┬──────────────────┐
│ 时间段      │ 标识     │ 典型活动          │
├─────────────┼──────────┼──────────────────┤
│ 06:00-09:00 │ early    │ 喂奶、换尿布、晨护│
│ 09:00-12:00 │ morning  │ 早教、户外、备餐  │
│ 12:00-15:00 │ noon     │ 午餐、午睡        │
│ 15:00-18:00 │ afternoon│ 游戏、外出、家务  │
│ 18:00-21:00 │ evening  │ 晚餐、洗澡、睡前  │
│ 21:00-06:00 │ night    │ 夜奶、哄睡（值班）│
└─────────────┴──────────┴──────────────────┘
```

**用户故事**
```
作为父母
我想看到今天所有需要完成的任务
以便提前规划和准备

作为妈妈
我想系统自动分配任务给爸爸
当我忙不过来时
```

#### 模块5：负载均衡引擎

**功能描述**
- 实时计算家庭成员的疲劳度
- 智能调整任务分配，避免过度劳累
- 提供负载均衡报告和建议

**疲劳度计算**

```python
# 疲劳度分数 = 基础分 + 连续工作惩罚 + 睡眠不足惩罚

基础分 = 今日已完成任务数 / 预分配任务数

连续工作惩罚：
- 连续2小时无休息：+0.2
- 连续4小时无休息：+0.5
- 夜间值班：+0.3

睡眠不足惩罚：
- 少于目标睡眠1小时：+0.2
- 少于目标睡眠2小时：+0.5
- 少于目标睡眠3小时：+1.0

疲劳度等级：
- 0.0-0.3：轻松 ✅
- 0.3-0.6：适中 ⚠️
- 0.6-0.8：疲劳 ⚠️⚠️
- 0.8+: 过度疲劳 ❌
```

**负载均衡指标**

```python
# 负担不均度 = |妈妈疲劳度 - 爸爸疲劳度|

均衡等级：
- 0.0-0.2：非常均衡 ✅
- 0.2-0.4：基本均衡 ⚠️
- 0.4+: 不均衡 ❌

优化策略：
- 将高疲劳度的人的未完成任务转移给低疲劳度的人
- 夜间值班强制交替
- 周末补休机制
```

**用户故事**
```
作为爸爸
我想看到自己的疲劳度
以便知道是否需要休息

作为夫妻
我们希望知道任务分配是否公平
以便调整和改进
```

#### 模块6：动态调整系统

**功能描述**
- 支持任务完成后的自动重分配
- 提供一键重新分配功能
- 应对临时情况变化

**触发场景**

##### 6.1 任务完成后
```
场景：妈妈提前完成了上午的任务
行为：
1. 标记该任务为已完成
2. 重新计算负载
3. 将下午的部分任务转移给妈妈
4. 更新看板
```

##### 6.2 临时有事
```
场景：爸爸突然需要加班
行为：
1. 一键点击"我忙不过来"
2. 系统将爸爸的未完成任务重新分配
3. 优先分配给妈妈，其次第三方
4. 实在无法完成的任务标记为"延期"
```

##### 6.3 宝宝特殊情况
```
场景：宝宝今天闹觉，特别粘人
行为：
1. 标记"宝宝特殊日"
2. 只保留必要任务（喂奶、换尿布）
3. 其他任务自动延期
4. 不影响疲劳度计算
```

##### 6.4 第三方协助
```
场景：奶奶今天来帮忙
行为：
1. 添加临时照护者
2. 将部分任务分配给奶奶
3. 降低夫妻双方疲劳度
```

**用户故事**
```
作为妈妈
当我突然有事需要外出时
我希望系统快速重新分配我的任务
以便爸爸知道需要做什么

作为父母
当宝宝不舒服时
我们希望简化今日任务
专注于照顾宝宝
```

## 三、用户流程设计

### 3.1 首次使用流程

```
1. 欢迎页面
   ↓
2. 创建宝宝档案
   - 输入基本信息
   - 设置生日（系统自动计算月龄）
   ↓
3. 添加家庭成员
   - 添加妈妈信息
   - 添加爸爸信息
   - 设置时间约束
   ↓
4. 查看今日任务
   - 系统自动生成今日SOP
   - 智能分配给父母
   ↓
5. 开始使用
```

### 3.2 日常使用流程

```
早晨（6:00-9:00）
- 打开应用，查看今日任务
- 完成任务，点击打勾
- 系统自动更新进度

上午（9:00-12:00）
- 继续完成分配的任务
- 如有特殊情况，点击调整

午间（12:00-15:00）
- 宝宝午睡时，查看今日总结
- 根据需要调整下午任务

下午+晚间
- 按计划执行
- 完成后标记

夜间
- 记录夜奶和哄睡情况
- 一天结束，查看总结报告
```

### 3.3 任务调整流程

```
情况1：任务完成后
1. 点击任务完成按钮
2. 系统自动记录完成时间和执行人
3. 重新计算负载，可能重新分配剩余任务

情况2：临时有事
1. 点击"我忙不过来"按钮
2. 选择受影响的时段
3. 系统重新分配该时段的任务
4. 确认调整

情况3：交换任务
1. 长按任务，拖动到另一人
2. 系统自动交换两人的任务
3. 重新计算疲劳度
```

## 四、非功能需求

### 4.1 性能要求
- 页面加载时间 < 2秒
- 任务操作响应 < 500ms
- 支持离线使用（未来）

### 4.2 可用性要求
- UI大屏优化，字体清晰
- 操作简单，步骤最少化
- 一键常用功能

### 4.3 可靠性要求
- 数据本地存储，不上传云端
- 定期自动备份
- 异常恢复机制

### 4.4 可扩展性要求
- 支持多宝宝（未来）
- 支持多第三方照护者
- 知识库可导入导出

## 五、成功指标

### 5.1 用户满意度
- 使用频率：每天打开次数 > 5
- 任务完成率：> 80%
- 用户留存：7日留存 > 70%

### 5.2 核心价值
- 负载均衡度：< 0.4（基本均衡）
- 睡眠充足率：> 90% 达到目标睡眠
- 育儿知识覆盖：100% 按时接种疫苗

### 5.3 用户体验
- 任务调整时间：< 30秒
- 首次使用上手时间：< 5分钟
- 零学习成本（无需阅读文档）
